# 插入外部引用（dmRef）

本文说明 **ietm-editor** 中「插入外部引用」的宿主接入方式：工具栏接管、写入 API、打开被引用 DM，以及与内置 mock 弹框的关系。

> 说明：本文件为独立说明，**不**写入仓库根目录 `README.md`。

---

## 功能概览

- 格式工具栏 **「插入外部引用」** 与「内部引用」同簇（`reference` 分组），内置按钮 id 为 `insertExternalRef`。
- 编辑器内以 **`dmRef`** 节点承载 S1000D 片段：`attrs.rawXml` 为完整 `<dmRef>…</dmRef>` 字符串（不含 SDK 扩展属性）；可选 `displayCode`、`refTargetId` 用于 UI 与宿主跳转，**导出 DM XML 时会写入 `<dmRef>` 根节点并 round-trip**（见第 8 节）。
- 未配置宿主回调时，点击按钮会打开 SDK 内置 **「引用出版物」** 弹框（mock 数据），确认后同样写入 `dmRef`。

---

## 1. 接管「插入」点击：`ToolbarConfig.onInsertExternalRefClick`

在 `createIETMEditor({ toolbar: { … } })` 或 `instance.setToolbarConfig({ … })` 中传入：

| 字段 | 说明 |
| --- | --- |
| `onInsertExternalRefClick` | 若设置：用户点击「插入外部引用」时**只**调用本回调，**不再**打开内置 mock 弹框。宿主自行打开选 DM / 选出版物 UI，确认后调用 `insertDmRefs`（见下文）。 |
| `onOpenExternalRefTarget` | 可选。用户在外部引用 Popover 中点击「打开」箭头时调用，用于新窗口加载目标 DM 等（见第 4 节）。 |

回调参数类型为 **`ToolbarItemContext`**（定义于 `src/types/toolbar.ts`）：

- `editor`：当前 TipTap `Editor`
- `editable`：是否可编辑
- `activeTabKey`：顶栏选项卡 `file` | `edit` | `insert`
- `formatBarLocked`：只读锁定时为 `true`（此时插入类按钮已禁用，一般不会进入回调）

---

## 2. 写入正文：`insertDmRefs` / `InsertDmRefPayload`

### 实例 API

`IETMEditorInstance` 提供：

```ts
insertDmRefs(items: InsertDmRefPayload[]): boolean;
```

- 成功插入返回 `true`；编辑器未就绪或当前位置不允许插入时返回 `false`。
- 建议在 `ready` 事件之后再依赖该 API（与 `insertImages` 等一致）。

### 载荷类型 `InsertDmRefPayload`

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `rawXml` | 是 | **完整** S1000D `<dmRef>…</dmRef>` XML 字符串（首尾空白会被 trim；SDK 会从根节点剥离扩展属性后存入节点）。 |
| `displayCode` | 否 | Popover 等 UI 展示编码；导出时写入 `<dmRef data-display-code="…">` 并随 DM 保存/重开恢复。 |
| `refTargetId` | 否 | 宿主跳转用稳定 ID（如 `dmInfoId`）；导出时写入 `<dmRef data-ref-target-id="…">` 并 round-trip。推荐用于 `onOpenExternalRefTarget`，勿再拼进 `displayCode`。 |

实现见 `src/lib/editor/insertDmRefs.ts`：内部将 payload 转为 TipTap `dmRef` 节点 JSON。

### 插入前校验（可选）

包入口导出：

- `canInsertDmRefIntoEditor(editor, item)`：当前 schema 与选区是否允许插入该条引用。
- `insertDmRefsIntoEditor(editor, items)`：已有 `Editor` 实例时可直接调用，等价于实例上的 `insertDmRefs`。
- `buildDmRefJsonFromPayload(item)`：仅需节点 JSON 时使用。

内置 mock 弹框确认时的典型写法可参考：`src/components/editor/ExternalRefPublicationModal.tsx`（`canInsertDmRefIntoEditor` → `insertDmRefsIntoEditor`）。

### `rawXml` 结构参考

宿主需按业务生成合法 `<dmRef>`。SDK 内置 mock 的生成逻辑可作为片段示例：`src/lib/editor/buildMockDmRefXml.ts`（`buildMockDmRefRawXml`），包含 `dmRefIdent`、`dmCode`、`issueInfo`、`language`、`dmRefAddressItems` / `dmTitle` 等。

## 3. 相关类型与导出

以下类型由包入口导出（名称以实际 `package.json` 包名为准，如 `@ccengine/ietm-editor`）：

- `ToolbarConfig`、`ToolbarItemContext`、`InsertDmRefPayload`
- `OpenExternalRefContext`（打开目标时用）
- `BuiltinToolbarItemId`（含 `insertExternalRef`）
- `CustomToolbarItem`、`ToolbarItemPlacement`（自定义按钮时用）

与外部引用展示解析相关的导出：

- `parseDmRefDisplayMeta`、`parseDmRefDisplayTitle`、`formatDmCodeLabel`
- `DmRefDisplayMeta`（类型）
- `openExternalRefPublication`（一般由 NodeView 内部调用；宿主侧主要配置 `onOpenExternalRefTarget`）

---

## 4. 打开被引用出版物：`onOpenExternalRefTarget`

用户悬停外部引用、在 Popover 中点击跳转箭头时，若配置了 `onOpenExternalRefTarget`，SDK 会解析节点中的 `rawXml`（及可选 `displayCode`）并调用宿主。

**`OpenExternalRefContext`** 字段包括：

- `editor`、`rawXml`
- `title`、`code`：展示用标题与编码（编码优先来自 `displayCode`，否则从 XML 解析 dmCode 拼接）
- `refTargetId`：宿主跳转主键（保存/重开后仍可用；未传时宿主可自行解析 `rawXml` / `dmCode`）
- `dmCode`：`dmCode` 元素各属性键值
- `issueInfo`、`language`

宿主典型实现：优先使用 `ctx.refTargetId` 打开目标 DM；否则根据 `rawXml` / `dmCode` 请求后端。未配置时 SDK 仅弹出演示提示（`src/lib/editor/openExternalRef.ts`）。

---

## 8. DM XML round-trip：SDK 扩展属性

保存/导出时，SDK 在 `<dmRef>` **根节点**写入下列属性（非 S1000D 标准；标准校验器可能提示 unknown attribute，ietm-editor 读写会保留）：

| XML 属性 | 节点 attr | 用途 |
| --- | --- | --- |
| `data-display-code` | `displayCode` | Popover 编码展示 |
| `data-ref-target-id` | `refTargetId` | 宿主「打开出版物」跳转 ID |

节点内存放的 `rawXml` 为**已剥离**上述属性后的 S1000D 片段；导出时由 `serializeDmRefToXml` 合并写回。

包入口导出常量：`DM_REF_DISPLAY_CODE_ATTR`、`DM_REF_TARGET_ID_ATTR`，以及 `serializeDmRefToXml`、`normalizeDmRefEditorAttrs`。

## 5. 隐藏内置按钮、改用自定义入口

若不需要内置图标，可：

```ts
hideBuiltinItems: ["insertExternalRef"],
customItems: [
  {
    id: "host-insert-dm-ref",
    title: "插入外部引用",
    placement: "reference",
    order: 10,
    onClick: ({ editor }) => {
      /* 与 onInsertExternalRefClick 相同：自选 UI + insertDmRefs */
    },
  },
],
```

## 6. 注意事项

1. **只读**：`formatBarLocked === true` 时格式栏插入按钮不可用。
2. **Schema**：当前描述类 / 故障隔离等 schema 须允许在光标位置插入 `dmRef`；否则 `canInsertDmRefIntoEditor` / `insertDmRefs` 会失败。
3. **内置弹框**：未传 `onInsertExternalRefClick` 时，行为为 `insertExternalRefFromSchema` → 打开内置 Modal（`src/lib/s1000d/descriptionSchemaInsert.ts`）。
4. **运行时改配置**：`instance.setToolbarConfig(config)`；传入 `null` 恢复默认；`destroy()` 时会重置工具栏 store。

## 7. 伪代码示例（宿主）

```ts
import {
  createIETMEditor,
  canInsertDmRefIntoEditor,
  type IETMEditorInstance,
  type InsertDmRefPayload,
} from "@ccengine/ietm-editor";

let instance: IETMEditorInstance;

instance = createIETMEditor({
  element: document.getElementById("editor-root")!,
  toolbar: {
    onInsertExternalRefClick: ({ editor }) => {
      openHostDmPicker({
        onPick: (dmRefXml: string, listCode?: string, dmInfoId?: string) => {
          const payload: InsertDmRefPayload = {
            rawXml: dmRefXml,
            displayCode: listCode,
            refTargetId: dmInfoId != null ? String(dmInfoId) : undefined,
          };
          if (!canInsertDmRefIntoEditor(editor, payload)) {
            notify("当前光标位置不能插入外部引用");
            return;
          }
          instance.insertDmRefs([payload]);
        },
      });
    },
    onOpenExternalRefTarget: async (ctx) => {
      const targetId = ctx.refTargetId ?? hostResolveIdFromDmCode(ctx);
      const xml = await hostFetchDmXml(targetId, ctx);
      hostOpenDmInNewTab(xml, ctx.code);
    },
  },
});

instance.on("ready", () => {
  /* 可安全使用 insertDmRefs 等 */
});
```

（`openHostDmPicker`、`notify`、`hostFetchDmXml`、`hostOpenDmInNewTab` 均为占位，由宿主实现。）
